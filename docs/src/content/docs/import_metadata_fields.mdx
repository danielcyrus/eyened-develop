---
title: Import Metadata Fields
description: Metadata fields available when importing data into the Eyened platform
---

When importing via the [Import API](/eyened-platform/importing_data), metadata is provided in a flat `data` object using **snake_case** field names. The tables below list the supported fields for each entity level.

## Patient

| API Field | Type | Description |
|----------|------|-------------|
| `patient_identifier` | str | Unique identifier for the patient (within a project) |
| `birth_date` | date | Patient's date of birth |
| `sex` | str | Patient's sex: `"M"` or `"F"` |

## Study

| API Field | Type | Description |
|----------|------|-------------|
| `study_date` | date | Date when the study was performed |
| `study_description` | str | Description of the study |

:::note
`StudyRound` exists in the database model but is not currently exposed in the Import API.
:::

## Series

| API Field | Type | Description |
|----------|------|-------------|
| `series_id` | str | Identifier used to match or create the series |
| `series_number` | int | Series number |
| `series_instance_uid` | str | Unique identifier for the series (DICOM) |

:::note
`StudyInstanceUid` exists in the database model but is not currently exposed in the Import API.
:::

## Image

| API Field | Type | Description |
|----------|------|-------------|
| `image` | str | **Required.** Path to the image file (relative to IMAGES_BASEPATH) |
| `modality` | str | **Required.** Type of image: `ColorFundus`, `OCT`, `OCTA`, etc. |
| `laterality` | str | Side of body: `"L"` or `"R"` |
| `height` | int | Number of rows (image height in pixels) |
| `width` | int | Number of columns (image width in pixels) |
| `depth` | int | Number of frames (e.g. B-scans in OCT) |
| `sop_instance_uid` | str | Unique identifier for SOP instance (DICOM) |
| `sop_class_uid` | str | Service-object pair class (DICOM) |
| `photometric_interpretation` | str | Pixel interpretation (RGB, MONOCHROME, etc.) |
| `samples_per_pixel` | int | Number of color components per pixel |
| `dicom_modality` | str | Equipment type: `OP`, `OPT`, `SC` |
| `etdrs_field` | str | ETDRS field position: `F1`–`F7` |
| `anatomic_region` | int | Body part examined |
| `angiography` | str | Angiography type indicator |
| `acquisition_date_time` | datetime | Date/time of acquisition |
| `pupil_dilated` | str | Whether pupil was dilated |
| `horizontal_field_of_view` | float | Field of view in millimeters |
| `resolution_axial` | float | Axial resolution (mm) |
| `resolution_horizontal` | float | Horizontal resolution (mm) |
| `resolution_vertical` | float | Vertical resolution (mm) |
| `attributes` | object | Key-value attributes (flexible schema) |

The `attributes` dictionary can hold arbitrary key-value pairs for image-level metadata not covered by the above fields.
